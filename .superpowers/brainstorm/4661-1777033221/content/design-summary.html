Stack

FrameworkTanStack Start (React) with static prerendering
ContentMarkdown files + YAML frontmatter, validated with Zod at build time
MathKaTeX via remark-math + rehype-katex
SearchPagefind — indexes static HTML, runs fully client-side
DeploymentStatic output → GitHub Pages / Netlify / Vercel

Content types & schema

Type Key extra fields Special UI
variety properties object: dimension, ambient_space, degree, singularities, genus, is_rational, kodaira_dimension, … Properties table + filterable browse page at /varieties
computation object (slug link), computes (text), techniques (list) Linked object, technique tags, filterable
counterexample refutes (what statement it disproves) Prominent "refutes" banner at top
Shared fields on all types: title · slug · field · tags · summary · related · references

Routes

/ — Homepage: search bar, type counts, foldable sidebar (fields + tags), recent entries
/examples — Full listing, filter by type/field/tag
/examples/$slug — Individual example (stable permalink)
/varieties — Filter varieties by property (dimension, ambient space, etc.)
/tags/$tag — All examples with a tag
/fields/$field — All examples in a field

Layout

Sidebar layout (option B) with a foldable sidebar containing field links and popular tags. Sidebar collapses on mobile and can be toggled on desktop. Main content area holds the search bar and example listings/detail pages.

Authoring workflow

Create a .md file in content/examples/ → build validates schema → static site rebuilds → new example is live with permalink. No CMS, no database. Git is the source of truth.
